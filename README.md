# Work-Lib

Work-Lib 是一个面向真实工作的个人知识库，用来沉淀企业文化、内部传播、员工活动、EAP 员工关爱、家庭日 / 开放日、AI 工作流和管理表达相关的内容资产。

它不是聊天记录整理站，也不是资料堆放区，而是一个可以直接阅读、查找、复用和转化的工作内容库。

网站地址：

https://goodnessy.github.io/work-lib/

---

## 网站定位

Work-Lib 的核心目标是：把日常工作中反复会用到的内容沉淀下来，把零散想法整理成完成稿、模板、清单和话术，在需要写方案、做汇报、推活动、写公众号、做复盘时，可以快速找到可用内容。

一句话概括：

> Work-Lib 是一个为真实工作服务的个人工作知识库。

---

## 主要栏目

### 首页

首页是 Work-Lib 的入口页，用来快速进入重点内容，包括本周精选、最近更新、栏目入口、标签入口和更新助手。首页不承担长文章阅读功能，只作为内容导航和重点推荐。

### 每日简报

每日简报不是普通新闻摘要，而是围绕近期重点工作形成的工作判断和行动提醒。

它重点关注：今天值得关注什么；这件事和实际工作有什么关系；可以转化成哪些行动；哪些内容可以继续沉淀进方案、话术或模板。

### 半月刊

半月刊用于沉淀阶段性专题内容。它不是每日内容合集，而是围绕一个阶段性主题做相对完整的拆解。

适合沉淀家庭日 / 开放日整体设计专题、HR 智能机器人落地专题、内部传播方法专题、AI 辅助企业文化工作的实践专题等内容。

### 灵感库

灵感库用于收集和改造活动创意、传播创意和内容选题。它不是简单的想法堆放区，而是一个可改造的案例库。

每条灵感应尽量回答：这个灵感是什么，适合解决什么问题，为什么有效，如何改造成实际工作场景，需要哪些物料，适合什么渠道，有哪些风险，如何复制使用。

### 方法论

方法论是 Work-Lib 中最偏工具箱的栏目。这里沉淀的是可以反复复用的流程、模板、清单和工作方法。

适合放入活动方案搭建流程、家庭日 / 开放日分流设计模板、活动复盘模板、公众号选题拆解方法、内部传播稿件检查清单、HR 智能机器人 FAQ 知识库字段模板、AI 辅助写方案工作流等内容。

### 管理视角

管理视角用于整理公开动态和公开表达，帮助内部传播学习管理表达。

目前只保留两部分：

- 动态：整理与管理表达、公司发展、组织信心、内部传播相关的公开事件。
- 发言：整理公开讲话、公开采访、公开署名文章、新闻报道中的公开引用或表达结构拆解。

管理视角不是公司新闻转载，也不是业务资料库。公开动态需要转化为内部传播能理解、能使用的表达；没有公开全文时，不虚构发言稿。

## 内容标准

Work-Lib 中的内容应当是完成稿，而不是聊天过程。

正式内容中应避免出现“这版修正”“下一版要”“我建议”“本页不做什么”“提醒”“待补充”“我们前面聊到”等过程语言。

每条内容入库前，需要判断：

1. 它是不是完成稿，而不是讨论过程；
2. 它能不能服务一个真实工作场景；
3. 它有没有可复制、可改写、可执行的部分；
4. 它和已有内容是否重复；
5. 它应该放在哪个栏目，而不是到处都放。

## 维护原则

- 宁可内容少一点，也要内容可用；
- 宁可慢一点，也不要把讨论过程直接入库；
- 同一观点不要在多个栏目反复出现；
- 栏目页负责导航，文章页负责深度内容；
- 管理视角必须基于公开可核验内容；
- 方法论必须能复制使用；
- 灵感库必须能转化为真实工作场景；
- 每日简报必须有判断、拆解、行动建议和可转化素材。

最终目标是让 Work-Lib 成为一个干净、稳定、可复用的个人工作知识库。
